# スケルトン: 専門エージェントのシステムプロンプト (prompt/prompt_specialist_{name}.py)

## 概要

専門エージェントのシステムプロンプトを定義するテンプレート。
各専門エージェントごとにファイルを作成し、ドメイン固有の指示を記述する。

## ファイル配置

`prompt/prompt_specialist_a.py`（専門エージェントAの場合）
`prompt/prompt_specialist_b.py`（専門エージェントBの場合）

## プロンプト方式の選択基準

| 要件 | 方式 | 理由 |
|:---|:---|:---|
| ビジネスルールや動的な情報をシステムプロンプトに埋め込みたい | パターンB（関数） | 実行時に情報を取得してプロンプトに反映する必要があるため |
| 上記以外（静的な指示のみ） | パターンA（定数） | 動的生成が不要なため |

**注意**: AIエージェントフレームワーク内のツールや制御機構（フック等）で実現できる機能は、システムプロンプトへの埋め込みではなく、それらの仕組みを使うことができる。

```python
# 4. 結果をユーザーに確認する
# 5. 出力を生成する

# ---- 重要な注意事項 ----
# - {業務ルールに基づく制約}
# - {データ検証ルール}

# ---- エラーハンドリング指示 ----
# - エラーが発生した場合はオーケストレーターに報告する

# ---- 対話ルール ----
# - 常に丁寧な日本語で応答する
"""
```

## カスタマイズガイド

1. **役割定義**: 専門エージェントの担当するドメインと責務を明確に記述する
2. **処理フロー**: 情報収集 → 検証 → 処理 → 確認 → 出力の流れを業務プロセスに沿って定義する
3. **ビジネスルール**: `knowledge/` 配下のポリシーモジュールから取得したルールを引数として受け取り埋め込む（パターンB）。ポリシー関数を呼び出す際はビジネスルール値（期限月数・金額閾値等）を `settings` から渡すこと（R9.12.3参照）
4. **引数とプレースホルダーの一対一対応**: テンプレート内の `{placeholder}` と関数シグネチャの引数は必ず一対一で対応させること。テンプレートに使われない引数・引数のないプレースホルダーは禁止（R9.12.2参照）
5. **重要な注意事項**: 一度に処理する単位、確認タイミング等、LLMの振る舞いを制御する具体的な指示を記述する
