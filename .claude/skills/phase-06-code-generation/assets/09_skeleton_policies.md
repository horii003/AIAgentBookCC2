# スケルトン: ビジネスルール・ポリシー (knowledge/{domain}_policies.py)

## 概要

専門エージェントが参照するビジネスルール・ポリシーをテキストとして返却するモジュール。
ドメインごとにファイルを作成し、業務ルールをマークダウン形式で定義する。

**重要**: このモジュールは他のモジュールに依存しない（純粋なテキスト返却のみ）。

## ファイル配置

`knowledge/{domain_a}_policies.py`（ドメインAの場合）
`knowledge/{domain_b}_policies.py`（ドメインBの場合）

## カスタマイズガイド

1. **ドメインごとのファイル作成**: 専門エージェントごとに対応するポリシーファイルを作成する
2. **チェック項目の定義**: 業務ルールに応じてチェック項目を定義する
3. **関数の引数**: 動的パラメータが不要な場合は引数なしの関数でもよい。動的パラメータが必要な場合は `**kwargs` ではなく明示的な名前付き引数として定義し、呼び出し側で `settings` から値を渡すこと（R9.12.3参照）
4. **段階的な対処手順**: 単純な却下ではなく、修正提案 → 理由確認 → 最終判断の流れを定義する
