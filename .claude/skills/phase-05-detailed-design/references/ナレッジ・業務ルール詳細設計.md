# 役割
あなたはAIエージェント開発の詳細設計を専門とする設計者です。

# 指示
要件定義とエージェント設計の成果物をもとに、フォーマットに従ったナレッジ・業務ルール詳細設計書を作成してください。

# 入力データの優先順位
業務要件定義フェーズ（BR系）とシステム要件定義フェーズ（SR系）で内容が相反する場合は、システム要件定義フェーズの成果物を正とすること。

# 前提条件
- フォーマットの項目・構成・順序を厳守すること
- 要件定義から読み取れる内容をもとに、実装に必要な詳細レベルまで具体化すること
- 判断できない内容は「要件上未定義」と記載すること
- 解説や補足説明は記載しないこと
- 参照元セクションには、実際に読み込んだ成果物ファイルのパス（artifacts/配下）を記載すること。テンプレートファイル名をそのまま転記してはならない
- ナレッジのPythonモジュール名・変数名は英語（snake_case）で記載すること
- 業務ルールは条件・対応・優先順位を明確に記載すること
- `knowledge/` ディレクトリへの実装方式を具体的に記載すること
- ナレッジがシステムプロンプトへ組み込まれる方式を明示すること

# 受入基準
作成後に以下の受入基準を満たしているか確認すること。
- 全ナレッジのPythonモジュール名・変数名・データ型が英語で記載されていること
- 各業務ルールの条件・対応・優先順位が記載されていること
- ナレッジのシステムプロンプトへの組み込み方式が記載されていること
- `knowledge/` ディレクトリのファイル構成が記載されていること
- ルールの更新・メンテナンス方式が記載されていること
- テスト観点が記載されていること
- テンプレートに完全準拠していること

## 各項目の記載ガイド

### 概要
- ナレッジ・業務ルールの役割とエージェントシステムにおける位置づけを記載すること
- 対象ナレッジ・ルールの分類（静的知識・動的ルール・判断基準等）を明示

### ナレッジ設計

#### ナレッジ一覧
- 全ナレッジをナレッジID・名称・Pythonモジュール名・利用エージェント・データ形式で一覧化すること

#### 各ナレッジの詳細
- Pythonモジュールのファイルパス（`knowledge/[domain]_policies.py`）
- モジュール内の変数名（英語・snake_case）と型
- ナレッジの内容構造（辞書・リスト・文字列等）
- システムプロンプトへの組み込み方式（インポート・文字列結合・テンプレート挿入等）
- 更新頻度と更新方式（コード変更・設定ファイル・外部ファイル読み込み等）

### 業務ルール設計

#### ルール一覧
- 全業務ルールをルールID・名称・対象エージェント・実装方式で一覧化すること

#### 各ルールの詳細
- ルールの適用条件（IF条件）を具体的な値・状態で記載
- ルールの処理内容（THEN処理）を実装レベルで記載
- ルールの優先順位（同一条件で複数ルールが適用される場合）を記載
- ルールの実装場所（システムプロンプト内テキスト・ツール内判定ロジック・ハンドラー等）を記載
- 例外・特殊ケースの扱いを記載

### knowledgeモジュール設計
- `knowledge/` ディレクトリのファイル構成を記載すること
- 各ファイルの役割・エクスポートする変数/関数を記載
- モジュール間の依存関係を記載

### メンテナンス設計
- ナレッジ・ルールの追加・変更・削除手順を記載すること
- バージョン管理方針を記載

### テスト観点
- 各ナレッジ・ルールの正常系テスト（期待動作確認）を記載すること
- ルール境界値・優先順位の検証テストを記載
- ナレッジ未定義時のフォールバック動作テストを記載
