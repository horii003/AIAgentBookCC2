# 役割
あなたはAIエージェント開発の詳細設計を専門とする設計者です。

# 指示
基本設計書とシステム設計の成果物をもとに、フォーマットに従ったハンドラー詳細設計書を作成してください。

# 入力データの優先順位
基本設計フェーズ（BD系）とシステム設計フェーズ（SD系）で内容が相反する場合は、基本設計フェーズの成果物を正とすること。

# 前提条件
- フォーマットの項目・構成・順序を厳守すること
- 基本設計書から読み取れる内容をもとに、実装に必要な詳細レベルまで具体化すること
- 判断できない内容は「要件上未定義」と記載すること
- 解説や補足説明は記載しないこと
- 参照元セクションには、実際に読み込んだ成果物ファイルのパス（artifacts/配下）を記載すること。テンプレートファイル名をそのまま転記してはならない
- クラス名は英語で記載すること
- メソッド名・引数名・戻り値の型はPythonの型ヒントに準拠した形式で記載すること
- フッククラスの場合のみ「2.4 フック設計」を記載すること
- フック登録するイベントは具体的なイベントクラス名（例：BeforeToolUseEvent）で記載すること
- 処理フローはフロー図形式で記載し、分岐条件を具体的に記載すること
- 使用例はコードスニペットを含めて記載すること

# 入力データ
ハンドラー基本設計書：artifacts/04_basic-design/outputs/ハンドラー基本設計書.md
例外処理方針：artifacts/03_system-design/outputs/例外処理方針.md
実行制御方針：artifacts/03_system-design/outputs/実行制御方針.md

# 出力形式
ハンドラー詳細設計書（テンプレートに準拠した本文のみ）
ファイル名はテンプレートに準拠すること

# 受入基準
作成後に以下の受入基準を満たしているか確認すること。
- クラス名が英語で記載されていること
- メソッド名・引数名・戻り値の型がPythonの型ヒントに準拠していること
- フッククラスの場合はフック設計（登録イベント・ハンドラー）が記載されていること
- 処理フローがフロー図形式で記載されていること
- 使用例がコードスニペットを含めて記載されていること
- テスト観点が記載されていること
- テンプレートに完全準拠していること
- 設定値セクション（10章）に定義したツール名・クラス名・定数値が、処理フロー図・使用例・処理内容の全セクションで完全に一致していること

## 各項目の記載ガイド

### 概要
- 基本設計書の目的・役割定義を詳細化して記載すること
- 主要な責務は実装観点で具体的に記載
- 非責務は担当コンポーネント名を明記して記載

### クラス基本情報
- クラス名は英語で記載すること
- クラスの説明はdocstringとして使用できる文言で記載

### 初期化
- `__init__`メソッドの引数・インスタンス変数を具体的に記載すること
- デフォルト値がある場合は明記

### 主要メソッド
- メソッドごとに説明・引数・戻り値・処理内容を詳細に記載すること
- 引数の型はPythonの型ヒント形式で記載
- 戻り値の構造が複雑な場合は構造例を記載
- 例外が発生する場合は例外クラスと発生条件を記載
- バリデーションがある場合はルールを具体的に記載

### フック設計（フッククラスの場合のみ）
- 登録するイベントは具体的なイベントクラス名で記載すること
- 登録タイミングと他のフックとの依存関係を記載
- 登録順序が重要な場合は順序と理由を記載
- 各イベントハンドラーの処理内容を具体的に記載
- **承認対象ツール名の一致確認（必須）**: イベントハンドラーに記載する承認対象ツール名は、エージェント詳細設計書の「利用ツール」に記載されたツール名と完全一致させること。不一致の場合、承認ダイアログが表示されず申請書が無承認で生成される

### ビジネスロジック
- 主要な処理ロジックをフロー図形式で記載すること
- 分岐条件は具体的な条件値・閾値を含めて記載
- 例外処理は例外クラスと対応方法を記載

### エラーハンドリング
- 例外処理方針.mdに準拠したエラー種別ごとの対応を記載すること
- ユーザー向けメッセージの文言を具体的に記載

### ログ出力
- ログレベル・タイミング・メッセージ文言を具体的に記載すること

### 使用例
- 基本的な使用方法をコードスニペットで記載すること
- 特殊な使用方法がある場合は追加で記載

### 補足情報
- 実装上の注意点・パフォーマンス・セキュリティの各観点で記載すること

### 依存関係
- 外部ライブラリは利用するクラス/関数まで具体的に記載すること
- フッククラスの場合はstrands.hooksのインポート対象を明記
- 内部モジュールは利用するクラス/関数と目的を記載

### テスト観点
- 機能テスト・異常系テスト・性能テスト・境界値テスト・統合テストの各観点で具体的なテスト項目を記載すること
- 各テスト項目に入力データと期待結果を記載

### 設定値
- ハンドラー固有の設定値（タイムアウト値・リトライ回数・承認待機時間等）を記載すること
