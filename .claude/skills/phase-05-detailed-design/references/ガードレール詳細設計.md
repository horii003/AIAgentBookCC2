# 役割
あなたはAIエージェント開発の詳細設計を専門とする設計者です。

# 指示
ガードレール処理方式設計・エージェント詳細設計・業務ルール定義をもとに、フォーマットに従ったガードレール詳細設計書を作成してください。

# 入力データの優先順位
ガードレール処理方式設計（SD-10）とエージェント詳細設計（DD-02）で内容が相反する場合は、エージェント詳細設計の成果物を正とすること。

# 前提条件
- フォーマットの項目・構成・順序を厳守すること
- 要件定義から読み取れる内容をもとに、実装に必要な詳細レベルまで具体化すること
- 判断できない内容は「要件上未定義」と記載すること
- 解説や補足説明は記載しないこと
- 参照元セクションには、実際に読み込んだ成果物ファイルのパス（artifacts/配下）を記載すること。テンプレートファイル名をそのまま転記してはならない
- Strands SDKの `BeforeModelInvokeEvent` / `AfterModelInvokeEvent` 等のフックイベントを利用する場合はイベントクラス名を具体的に記載すること
- ガードレール種別（入力フィルター・出力フィルター・ツール実行ガード）ごとに設計を分けること
- ブロック条件は具体的な判定ロジック（正規表現・キーワードリスト・閾値等）で記載すること
- フォールバック応答はユーザー向けの実際のメッセージ文言で記載すること

# 受入基準
作成後に以下の受入基準を満たしているか確認すること。
- ガードレール種別（入力フィルター・出力フィルター・ツール実行ガード）が区別されていること
- ブロック条件が実装レベルで具体的に記載されていること（曖昧な表現がないこと）
- フォールバック応答のメッセージ文言が具体的に記載されていること
- 適用対象エージェントが明示されていること
- ガードレール通過後の処理フローが記載されていること
- エラーハンドリングが記載されていること
- テスト観点が記載されていること
- テンプレートに完全準拠していること

## 各項目の記載ガイド

### 概要
- ガードレールの目的とStrands SDKにおける実装方式を記載すること
- 適用対象エージェント・ガードレール種別の全体像を明示

### ガードレール一覧
- 設計書に含まれる全ガードレールを一覧化すること
- ガードレールID・種別・対象エージェント・概要を記載

### 入力フィルター設計
- ユーザー入力の検査タイミング・検査方法・ブロック条件を記載すること
- ブロック対象のパターン（禁止キーワード・正規表現等）を具体的に記載
- ブロック時のフォールバック応答メッセージ文言を記載
- Strands SDKフック（`BeforeModelInvokeEvent`等）を利用する場合は実装方式を記載

### 出力フィルター設計
- LLM応答の検査タイミング・検査方法・フィルター条件を記載すること
- フィルター対象パターン（機密情報・不適切表現等）を具体的に記載
- フィルター後の応答修正・置換ロジックを記載

### ツール実行ガード設計
- ツール実行前の承認要否判定ロジックを記載すること（自律度・権限定義.mdのマトリクスに基づく）
- 承認が必要なツール・エージェントの組み合わせを一覧化
- 人間承認フックとの連携方式を記載

### 判定ロジック詳細
- 各ブロック/フィルター条件の判定アルゴリズムを実装レベルで記載すること
- 判定に使用するデータソース（禁止リスト・ルールテーブル等）のファイルパスと形式を記載

### フォールバック応答設計
- ガードレール発動時のユーザー向けメッセージ文言一覧を記載すること
- ガードレール種別・発動条件・メッセージ文言を対応付けて記載

### エラーハンドリング
- ガードレール処理中の例外（検査エンジンエラー等）の対応を記載すること

### ログ出力
- ガードレール発動・通過のログレベル・タイミング・メッセージ文言を記載すること

### テスト観点
- 各ガードレールの正常系（通過）・異常系（ブロック）テストを記載すること
- 境界値テスト（ブロック判定の閾値付近）を記載
