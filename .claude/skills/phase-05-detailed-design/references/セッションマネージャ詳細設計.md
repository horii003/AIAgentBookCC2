# 役割
あなたはAIエージェント開発の詳細設計を専門とする設計者です。

# 指示
基本設計書とシステム設計の成果物をもとに、フォーマットに従ったセッションマネージャ詳細設計書を作成してください。

# 入力データの優先順位
基本設計フェーズ（BD系）とシステム設計フェーズ（SD系）で内容が相反する場合は、基本設計フェーズの成果物を正とすること。

# 前提条件
- フォーマットの項目・構成・順序を厳守すること
- 基本設計書から読み取れる内容をもとに、実装に必要な詳細レベルまで具体化すること
- 判断できない内容は「要件上未定義」と記載すること
- 解説や補足説明は記載しないこと
- 参照元セクションには、実際に読み込んだ成果物ファイルのパス（artifacts/配下）を記載すること。テンプレートファイル名をそのまま転記してはならない
- クラス名・メソッド名・引数名は英語で記載すること
- 型はPythonの型ヒントに準拠した形式で記載すること
- セッションの永続化ファイルパスはセッション管理方針.mdの定義に従うこと
- Strands SDKの `ConversationManager` / `FileConversationManager` の使用仕様を明示すること
- 会話履歴の保存・復元・削除の各ライフサイクルを具体的に記載すること

セッションマネージャ基本設計書：artifacts/04_basic-design/outputs/セッションマネージャ基本設計書.md

# 出力形式
セッションマネージャ詳細設計書（テンプレートに準拠した本文のみ）

# 受入基準
作成後に以下の受入基準を満たしているか確認すること。
- クラス名・メソッド名が英語で記載されていること
- セッションライフサイクル（生成・保存・復元・削除）が実装レベルで記載されていること
- 永続化ファイルパスの生成ルールが具体的に記載されていること
- Strands SDKのConversationManager利用仕様が明示されていること
- 会話履歴のウィンドウサイズ設定が記載されていること
- エラーハンドリングが例外処理方針.mdに準拠していること
- テスト観点が記載されていること
- テンプレートに完全準拠していること

## 各項目の記載ガイド

### 概要
- セッションマネージャの目的とStrands SDKとの関係を記載すること
- 管理対象（会話履歴・セッションID・ユーザー状態等）を明示

### クラス基本情報
- クラス名は英語で記載すること
- 基底クラス・継承元を記載
- コンストラクタの引数・インスタンス変数を具体的に記載

### セッションライフサイクル設計
- セッション生成・保存・復元・削除の各フローをフロー図形式で記載すること
- 各フェーズのトリガー条件と事後処理を記載
- 同一ユーザーの複数セッション管理方針を記載

### 永続化設計
- ファイルパス生成ルール（ディレクトリ構造・ファイル名規則）を記載すること
- シリアライズ形式（JSON等）とエンコード仕様を記載
- ConversationManagerの設定パラメータ（storage_dir・window_size等）を記載

### Strands SDK連携設計
- `ConversationManager` / `FileConversationManager` のインスタンス生成方法を記載すること
- エージェントへの渡し方（`conversation_manager` パラメータ）を記載
- ウィンドウサイズ（オーケストレーター/専門エージェント別）の設定値と根拠を記載

### 主要メソッド設計
- メソッドごとに説明・引数・戻り値・処理フローを詳細に記載すること
- 例外が発生する場合は例外クラスと発生条件を記載

### エラーハンドリング
- ファイル読み書きエラー・セッション不整合等のエラー処理を記載すること
- 例外処理方針.mdに準拠したエラー種別ごとの対応を記載

### ログ出力
- セッション操作（生成・保存・復元・削除）ごとのログレベル・タイミング・メッセージ文言を記載すること

### テスト観点
- セッション生成・保存・復元・削除の正常系テストを記載すること
- ファイル不存在・破損データ等の異常系テストを記載
