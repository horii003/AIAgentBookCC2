## ターゲットユーザー
社内の一般社員（申請ルールや申請方法に詳しくない場合がある）

## 要求事項
- RQ-01: 社員が申請内容を入力すると必要な申請種別を自動的に判断できること
- RQ-02: 使用すべき申請書と申請先を明示的に提示できること
- RQ-03: 申請に必要な不足情報を対話で確認・収集できること
- RQ-04: 収集した情報をもとに申請書を自動作成できること
- RQ-05: 申請ミスや差し戻しを削減できること

| ----- | -------- | --------- | ------ | ------ |

## 対象申請種別

本システムが対応する申請種別は以下の2種類のみとする。

| 申請種別 | 申請書名 |
| -------- | -------- |
| 交通費精算申請 | 交通費精算申請書 |
| 経費精算申請 | 経費精算申請書 |

申請内容から申請種別を判断できない場合は、「交通費精算申請」「経費精算申請」の選択肢を社員に提示し、社員の選択に従って対応する申請フローへ誘導する。

## 業務種別ごとの業務ルール

### 交通費精算申請の業務ルール

| ID | 業務ルール | 詳細 | 優先度 |
| -- | ---------- | ---- | ------ |
| TRF-01 | 交通費精算申請の申請対象 | 業務上発生した交通費（電車・バス・タクシー・飛行機の4種類）が対象。申請は経費発生日から3ヶ月以内であること | High |
| TRF-02 | 交通費精算申請の申請先 | 申請先は要件上未定義 | High |
| TRF-03 | 交通費精算申請の必須項目 | 移動日・出発地・目的地・交通手段・費用・業務目的を一区間ずつ収集する | High |
| TRF-04 | 交通費精算申請の運賃計算 | 運賃データから交通費を自動計算する（電車は経路テーブル検索、バス・タクシー・飛行機は固定運賃） | High |
| TRF-05 | 交通費精算申請の上長承認要件 | 交通費が10,000円を超える場合は事前に上長の承認が必要 | High |
| TRF-06 | 交通費精算申請の駅名正規化 | 駅名を正規化して取り扱う（例：「渋谷駅」→「渋谷」） | Medium |

### 経費精算申請の業務ルール

| ID | 業務ルール | 詳細 | 優先度 |
| -- | ---------- | ---- | ------ |
| EXP-01 | 経費精算申請の申請対象 | 業務上発生した経費のうち、事務用品費・宿泊費・資格精算費・その他経費の4区分が対象。申請は経費発生日から3ヶ月以内であること | High |
| EXP-02 | 経費精算申請の申請先 | 申請先は要件上未定義 | High |
| EXP-03 | 経費精算申請の必須項目 | 購入日・店舗名・品目・金額・業務目的・経費区分を品目ごとに収集する | High |
| EXP-04 | 経費精算申請の情報抽出 | 領収書画像から経費情報（購入日・店舗名・品目・金額）を品目ごとに自動抽出する | High |
| EXP-05 | 経費精算申請の経費区分判断 | 品目から経費区分（事務用品費・宿泊費・資格精算費・その他経費）を自動判断する | High |
| EXP-06 | 経費精算申請の上長承認要件 | 経費が5,000円を超える場合は事前に上長の承認が必要 | High |
