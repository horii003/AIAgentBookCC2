## 1. 概要

### 1.1 コンポーネントの目的

HitL承認（OK）後に収集済み申請情報（D-007）を申請種別に対応する申請書テンプレート（DATA-002: `data/templates/`）の各セルにマッピングし、申請書ドラフト（Excelファイル）を生成して `outputs/` ディレクトリに保存する。交通費精算用（AG-002）と経費精算用（AG-003）の2種類の関数を提供する。

### 1.2 主要な責務

1. **入力バリデーション**: `TransportExpenseFormInput` / `GeneralExpenseFormInput` Pydanticモデルによる全必須項目の型・形式チェック
2. **テンプレート読み込み**: DATA-002 の申請種別に対応する Excel テンプレートファイルを `openpyxl` でロードする
3. **セルマッピング**: 申請者名・申請日・移動区間/経費リスト・合計金額を定義されたセル位置に書き込む
4. **Excel ファイル出力**: 生成した申請書ドラフトを `outputs/` ディレクトリにタイムスタンプ付きのファイル名で保存する
5. **補完禁止の保証**: 収集済み申請情報に存在しないフィールドを推測・補完しない（BRL-04準拠）

## 3. generate_transport_expense_formツール設計

### 3.1 基本情報

#### ツール名（関数名）
`generate_transport_expense_form`

#### 説明（docstring）
```
交通費精算申請書ドラフト（Excelファイル）を生成して outputs/ ディレクトリに保存する。
HitL承認（OK）後にのみ呼び出すこと。収集済み情報に含まれないフィールドは補完しない（BRL-04）。

Args:
    sections (list[dict]): 移動区間リスト（1件以上）。各要素は以下のキーを持つ辞書:
        - travel_date (str): 移動日（YYYY-MM-DD形式）
        - departure (str): 出発地（正規化済み）
        - destination (str): 目的地（正規化済み）
        - transport_type (str): 交通手段（電車/バス/タクシー/飛行機）
        - fare (int): 交通費（円、非負整数）
        - purpose (str): 業務目的（500文字以内）
    total_fare (int): 交通費合計（円、非負整数）。各区間fareの合計。

Returns:
    dict: 以下のキーを持つ辞書:
        - file_path (str): 生成した申請書ドラフトのファイルパス（outputs/ 配下）
        - status (str): 生成結果ステータス（"success" または "failure"）
        エラー時: {"success": False, "message": "エラーメッセージ文字列"}

Raises:
    ValidationError: 入力バリデーション失敗時（必須項目欠落等）
    Exception: 上記以外の想定外例外
```

---

### 3.2 インターフェース設計

#### 3.2.1 入力

**パラメータ定義**:

| 引数名 | 型 | 説明 | デフォルト値 |
|--------|-----|------|--------------|
| `sections` | `list[dict]` | 移動区間リスト（1件以上） | なし（必須） |
| `total_fare` | `int` | 交通費合計（円、非負整数） | なし（必須） |

**各要素のフィールド（sections）**:

| フィールド名 | 型 | 説明 | デフォルト値 |
|------------|-----|------|--------------|
| `travel_date` | `str` | 移動日（YYYY-MM-DD形式） | なし（必須） |
| `departure` | `str` | 出発地（正規化済み） | なし（必須） |
| `destination` | `str` | 目的地（正規化済み） | なし（必須） |
| `transport_type` | `str` | 交通手段（電車/バス/タクシー/飛行機） | なし（必須） |
| `fare` | `int` | 交通費（円、非負整数） | なし（必須） |
| `purpose` | `str` | 業務目的（500文字以内） | なし（必須） |

##### バリデーションルール詳細

| フィールド | ルール | エラーメッセージ | 例 |
|-----------|--------|----------------|-----|
| `sections` | 1件以上 | `"移動区間は1件以上必要です。"` | `[]` → エラー |
| `sections[i].fare` | 非負整数 | `"金額は0以上の整数で入力してください"` | `"-100"` → エラー |
| `sections[i].purpose` | 非空・500文字以内 | `"この項目は必須です。"` | `""` → エラー |
| `total_fare` | 非負整数 | `"金額は0以上の整数で入力してください"` | `"-1"` → エラー |

#### 3.2.2 出力

**戻り値の型**: `dict[str, str]`

**説明**: 生成した申請書ドラフトのファイルパスと生成結果ステータスを含む辞書

**構造**:
```python
{
    "file_path": str,  # 生成ファイルパス（例: "outputs/交通費精算申請書_20260703120000.xlsx"）
    "status": str,     # "success" または "failure"
}
```

#### 3.2.3 例外

| 例外 | 発生条件 |
|------|---------|
| `ValidationError` | 入力バリデーション失敗（必須項目欠落・形式不正等）。エラー時戻り値: `{"success": False, "message": "エラーメッセージ文字列"}` |
| `Exception` | 上記以外の想定外例外。エラー時戻り値: `{"success": False, "message": "エラーメッセージ文字列"}` |

> **注記**: ファイル保存メソッド（_save_file）は IOError・PermissionError・Exception の3種類を個別に処理し、タプル形式 `(False, "エラーメッセージ文字列")` を返す。全エラーログには `extra={"file_path": file_path}` を付加する。

### 3.3 ビジネスロジック

#### 3.3.1 処理フロー

```
開始
  ↓
Step 0: invocation_state から申請者名・申請日を取得する
  applicant_name = tool_context.invocation_state["applicant_name"]
  application_date = tool_context.invocation_state["application_date"]
  ↓
TransportExpenseFormInput Pydanticモデルで入力バリデーション
  - sections: list[TransportSection]（各区間の全フィールドをバリデーション）
  - total_fare: parse_amount（非負整数）
  ↓
バリデーション失敗?
  Yes → {"success": False, "message": "申請情報に不備があります。{フィールド別メッセージ}"} を返す（EX-01）
  No ↓
_load_template("交通費精算申請書テンプレート.xlsx") を呼び出す
  ↓
テンプレートファイル不存在 → {"success": False, "message": "申請書テンプレートが見つかりません。担当部門にお問い合わせください。"} を返す（EX-06）
テンプレート読み込み失敗 → {"success": False, "message": "申請書テンプレートの読み込みに失敗しました。担当部門にお問い合わせください。"} を返す（EX-06）
  ↓
ワークシートを取得（ws = wb.active）
  ↓
固定セルへの書き込み:
  - ws["B3"] = applicant_name
  - ws["B4"] = application_date.strftime("%Y-%m-%d")
  ↓
移動区間リストのセルマッピング（i = 0 〜 n-1）:
  - ws[f"A{7+i}"] = i + 1（No）
  - ws[f"B{7+i}"] = sections[i].travel_date.strftime("%Y-%m-%d")
  - ws[f"C{7+i}"] = sections[i].departure
  - ws[f"D{7+i}"] = sections[i].destination
  - ws[f"E{7+i}"] = sections[i].transport_type
  - ws[f"F{7+i}"] = sections[i].fare
  - ws[f"G{7+i}"] = sections[i].purpose
  - ws[f"H{7+i}"] = ""（承認状況：空欄）
  ↓
合計金額セルへの書き込み:
  - ws[f"H{7+n+2}"] = total_fare
  ↓
出力ファイルパスを生成（タイムスタンプ方式）:
  - output_path = f"outputs/交通費精算申請書_{datetime.now().strftime('%Y%m%d%H%M%S')}.xlsx"
  ↓
wb.save(output_path)
  ↓
書き込み失敗?
  PermissionError → (False, "ファイルへの書き込み権限がありません。") を返す（EX-06）
  IOError → (False, "ファイルの書き込みに失敗しました。") を返す（EX-06）
  Exception → (False, "ファイル保存中に予期しないエラーが発生しました。") を返す（EX-08）
  ↓
{"file_path": output_path, "status": "success"} を返す
```

#### 3.3.2 データの検証方法

**Pydanticモデル**: `TransportExpenseFormInput`

**検証内容**:
- `sections`: `list[TransportSection]` として各区間の全フィールドをバリデーション
- `total_fare`: `parse_amount` バリデータで非負整数に変換
- `sections` リストが1件以上であることを `model_validator(mode="after")` で確認

> **注記**: `applicant_name`・`application_date` は `tool_context.invocation_state` から取得するため、Pydanticモデルのバリデーション対象には含まない。

### 3.4 エラーハンドリング

#### 3.4.1 エラー処理フロー

```
エラー発生
  ↓
エラー種別の判定
  ↓
ValidationError（入力バリデーション失敗）
  → EX-01として分類
  → 日本語エラーメッセージを生成
  → {"success": False, "message": "申請情報に不備があります。{フィールド別メッセージ}"} を返す
  → ErrorHandlerに渡す
  → セッション状態はWAITINGのまま（社員に再入力を促す）

Exception（上記以外の例外）
  → EX-08として分類
  → 再試行なし
  → {"success": False, "message": "申請処理中にエラーが発生しました。しばらく経ってから再度お試しいただくか、担当部門にお問い合わせください。"} を返す
  → エスカレーション案内・セッションTERMINATED
  → ErrorHandlerに渡す

【ファイル保存メソッド（_save_file）のエラー処理】
  PermissionError
    → (False, "ファイルへの書き込み権限がありません。") を返す（EX-06）
    → ログ出力: logger.error(..., extra={"file_path": file_path})
  IOError
    → (False, "ファイルの書き込みに失敗しました。") を返す（EX-06）
    → ログ出力: logger.error(..., extra={"file_path": file_path})
  Exception
    → (False, "ファイル保存中に予期しないエラーが発生しました。") を返す（EX-08）
    → ログ出力: logger.error(..., extra={"file_path": file_path})
```

#### 3.4.2 エラーメッセージの詳細

| エラー種別 | 発生条件 | 対応 | ユーザー向けメッセージ |
|-----------|---------|------|---------------------|
| EX-01（入力例外） | バリデーション失敗（必須項目欠落等） | 情報収集ステップへ戻す | `{"success": False, "message": "申請情報に不備があります。{フィールド別メッセージ}"}` |
| EX-06（テンプレートなし） | DATA-002 ファイルが存在しない | エスカレーション案内・TERMINATED | `{"success": False, "message": "申請書テンプレートが見つかりません。担当部門にお問い合わせください。"}` |
| EX-06（ファイル書き込み権限なし） | PermissionError 発生時 | エスカレーション案内・TERMINATED | `(False, "ファイルへの書き込み権限がありません。")` |
| EX-06（ファイル書き込み失敗） | IOError 発生時 | エスカレーション案内・TERMINATED | `(False, "ファイルの書き込みに失敗しました。")` |
| EX-08（ファイル保存想定外） | ファイル保存メソッド内の Exception | エスカレーション案内・TERMINATED | `(False, "ファイル保存中に予期しないエラーが発生しました。")` |
| EX-08（想定外） | 上記以外の例外 | エスカレーション案内・TERMINATED | `{"success": False, "message": "申請処理中にエラーが発生しました。しばらく経ってから再度お試しいただくか、担当部門にお問い合わせください。"}` |

```
合計金額セルへの書き込み:
  - ws[f"H{7+n+2}"] = total_amount
  ↓
出力ファイルパスを生成（タイムスタンプ方式）:
  - output_path = f"outputs/経費精算申請書_{datetime.now().strftime('%Y%m%d%H%M%S')}.xlsx"
  ↓
wb.save(output_path)
  ↓
書き込み失敗?
  PermissionError → (False, "ファイルへの書き込み権限がありません。") を返す（EX-06）
  IOError → (False, "ファイルの書き込みに失敗しました。") を返す（EX-06）
  Exception → (False, "ファイル保存中に予期しないエラーが発生しました。") を返す（EX-08）
  ↓
{"file_path": output_path, "status": "success"} を返す
```

## 6. 補足情報

### 6.1 実装上の注意点

1. **出力ファイルパスのタイムスタンプ自律生成**
   - ファイル名は `datetime.now().strftime("%Y%m%d%H%M%S")` をもとにツール内部で生成する。LLMのパラメータ（ツール引数）として受け取ってはならない（基本設計 14.3節準拠）

2. **テンプレートをコピーして書き込む**
   - `openpyxl.load_workbook()` でテンプレートをロードしてから書き込む。テンプレートファイル自体を直接変更しない

3. **HitL承認前呼び出し禁止**
   - HitL承認（OK）受信後のみ呼び出す（実行制御方針 6.2節・11.2節準拠）。エージェントの呼び出しロジックで保証すること

4. **再試行なし**
   - テンプレートファイル参照失敗は構成上の問題であるため再試行せず即時エスカレーション案内（実行制御方針 7.1節準拠）

5. **`@tool` デコレータの使用**
   - strands-agents v1.25.0 の `@tool` デコレータを使用して各関数を定義する

---

### 6.2 パフォーマンス考慮事項

1. **テンプレートのコピー方式**
   - テンプレートファイルを `openpyxl` でロードして書き込む方式は、区間/経費件数に依存するが通常の申請ではミリ秒オーダーで完了する

2. **ファイルパスの自律生成**
   - タイムスタンプ方式により並列セッション間でのファイル名衝突リスクを低減する

---

### 6.3 セキュリティ考慮事項

1. **補完禁止（BRL-04）**
   - 収集済み申請情報に存在しないフィールドは推測・補完せず、空欄またはエラーとする

2. **ファイル直接参照（外部API不使用）**
   - RAGを使用せず Excel テンプレートファイルを直接ローカルから参照する

3. **出力先ディレクトリ固定**
   - 出力先は `outputs/` ディレクトリに固定し、ディレクトリトラバーサルを防止する

## 8. テスト観点

### 8.1 機能テスト
- `tool_context.invocation_state` に `applicant_name`・`application_date` が設定されている状態で呼び出した場合に、申請書テンプレートの正しいセル（B3: 申請者名・B4: 申請日・A7〜H7+n: 各区間・H{7+n+2}: 合計金額）に書き込まれること（BRL-04：補完なし）
- 経費精算申請情報（複数経費）を渡した場合に、申請書テンプレートの正しいセルに書き込まれること
- 生成されたファイルが `outputs/` ディレクトリに `交通費精算申請書_YYYYMMDDHHMMSS.xlsx` の形式で保存されること
- 正常終了時の戻り値が `{"file_path": output_path, "status": "success"}` の形式であること

### 8.2 異常系テスト
- テンプレートファイルが存在しない場合に `{"success": False, "message": "申請書テンプレートが見つかりません。担当部門にお問い合わせください。"}` が返されること（EX-06）
- ファイル書き込み権限がない場合（PermissionError）にファイル保存メソッドが `(False, "ファイルへの書き込み権限がありません。")` を返し、ログに `extra={"file_path": file_path}` が含まれること（EX-06）
- ファイル書き込み失敗時（IOError）にファイル保存メソッドが `(False, "ファイルの書き込みに失敗しました。")` を返し、ログに `extra={"file_path": file_path}` が含まれること（EX-06）
- `sections` が空リストの場合に `ValidationError`（EX-01）が発生し `{"success": False, "message": ...}` が返されること
- `total_fare` が負数の場合に `ValidationError`（EX-01）が発生し `{"success": False, "message": ...}` が返されること
- 想定外例外発生時に `{"success": False, "message": "申請処理中にエラーが発生しました。..."}` が返されること（EX-08）

### 8.3 性能テスト
- 10区間の交通費精算申請書生成が3秒以内に完了すること
- 20件の経費精算申請書生成が3秒以内に完了すること

### 8.4 境界値テスト
- `sections` が1件（最小値）の場合に正常生成されること
- `invocation_state["applicant_name"]` が500文字の場合に正常生成されること
- `total_fare` が0円の場合に正常生成されること
