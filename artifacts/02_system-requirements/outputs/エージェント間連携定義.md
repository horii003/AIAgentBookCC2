# エージェント間連携一覧（協調・委譲・イベント）

## 1. 連携ポリシー（概要）
- 連携方式：同期（リクエスト/レスポンス型）
- 連携単位：タスク（業務フロー単位のステージ委譲）
- 失敗時方針：人へエスカレーション（エージェントが判断不能の場合は社員に確認を促す）
- 相互呼び出し制約：循環呼び出し禁止、最大委譲深度：1（AG-001→AG-002 または AG-001→AG-003）、タイムアウト：要件上未定義

## 3. 連携種別の定義（記入ルール）

### 3.1 協調（Collaboration）
- 目的：複数エージェントが**並行/分担**して成果を統合
- 本システムにおける適用：なし（申請フローはAG-001による振り分け後、各専門エージェントが全フローを自己完結する）

### 3.2 委譲（Delegation）
- 目的：起点エージェントがサブタスクを**丸投げ**して結果を受け取る
- 本システムにおける適用：LNK-001、LNK-002

### 3.3 イベント（Event-driven）
- 目的：状態変化（例：申請書生成完了）を発火し、購読側が処理
- 本システムにおける適用：なし（申請フローは同期委譲で構成し、非同期イベントは使用しない）

## 4. 連携インタフェース定義

### 4.1 連携ID：LNK-001

#### 4.1.1 トリガー条件
- 発火条件：AG-001がJD-01（申請種別特定）を適用し、申請種別が「交通費精算申請」と確定した時点
- 事前条件（Pre-condition）：申請者名と申請内容テキスト（D-001）が入力されており、社内申請ルール（DATA-001）が参照可能な状態であること；申請種別が「交通費精算申請」に確定していること
- 事後条件（Post-condition）：AG-002が交通費精算申請の全フローを完了し、申請書ドラフト（D-009）とチェック結果（D-010）が社員に提示されていること

#### 4.1.2 入力スキーマ（構造化推奨）
```json
{
  "request_id": "string",
  "task": "交通費精算申請フロー全体",
  "context": {
    "applicant_name": "string",
    "application_type": "交通費精算申請",
    "application_form_name": "string",
    "application_destination": "要件上未定義",
    "judgment_basis": "string（参照ルール名）"
  },
  "constraints": {
    "bulk_confirmation": true,
    "no_inference": true,
    "hitl_required_before_excel": true,
    "deadline_check_required": true
  },
  "approval_required": false
}
```

#### 4.1.3 出力スキーマ（構造化推奨）
```json
{
  "request_id": "string",
  "status": "completed | error",
  "result": {
    "application_type": "string",
    "draft_file": "string（ExcelファイルパスまたはID）",
    "check_status": "合格 | 不備あり",
    "issues": []
  },
  "error": {
    "code": "string",
    "message": "string"
  }
}
```

#### 4.1.4 エラー処理
- 収集タイムアウト：AG-001へエスカレーション → 社員に再入力を促す
- HitL承認でキャンセル選択：処理を中断し社員に通知
- 申請書テンプレート参照不能：AG-001へエスカレーション → 社員に状況を通知
