## 2. データ詳細定義（データID単位：任意）

### 2.1 データID：DATA-001（データ名：社内申請ルール）
- 種別：ファイル
- 概要：申請種別ごとの適用条件・申請先・必須項目・チェック基準等を定義した社内申請ルール情報
- 形式（拡張子/スキーマ）：要件上未定義
- 保管場所：ファイルストレージ
- 更新頻度：不定期（申請ルール改定時）
- 参照方法（API/検索/ダウンロード等）：ファイル直接参照（エージェントが起動時または参照時にロード）
- データ分類・機密区分：D-1（社内）/ 社内
- マスキング要件：なし
- 保持期間：要件上未定義
- 品質要件（欠損率/更新遅延/正確性など）：要件上未定義
- 監査要件：参照ログを記録する

#### スキーマ（要件定義レベル：代表項目のみ）
| No | 項目名 | 型（概略） | 必須 | 説明 | 例 |
|---:|---|---|---|---|---|
| 1 | 申請種別名 | string | ○ | 申請種別の名称 | 交通費精算申請 |
| 2 | 申請種別適用条件 | string | ○ | 当該申請種別が必要となる条件 | 要件上未定義 |
| 3 | 申請先 | string | ○ | 申請書の提出先 | 要件上未定義 |
| 4 | 申請書必須項目 | object | ○ | 申請書作成に必要な必須入力項目の定義 | 要件上未定義 |

## 3. AIエージェント向け追加項目（ファイル参照系データに推奨）

### DATA-001（社内申請ルール）

| 項目 | 内容 |
|---|---|
| 参照対象範囲 | 社内申請ルール全体（申請種別・適用条件・申請先・必須項目・チェック基準） |
| 参照方針 | 起動時にロードしてエージェントコンテキストに保持する |
| メタデータ | 申請種別区分、ルール改定日 |
| 根拠提示 | 適用ルール名・条件を提示（Lv2：参照元提示） |
| 更新検知 | 要件上未定義 |

### DATA-002（申請書テンプレート）

| 項目 | 内容 |
|---|---|
| 参照対象範囲 | 申請種別（交通費精算申請・経費精算申請）に対応するExcelテンプレートファイル |
| 参照方針 | 申請書生成ツール（TOOL-002）が申請種別に対応するテンプレートファイルを参照する |
| メタデータ | 申請種別区分、テンプレート改定日 |
| 根拠提示 | なし（生成に使用するテンプレートとして参照） |
| 更新検知 | 要件上未定義 |

### DATA-003（電車経路・運賃データ）

| 項目 | 内容 |
|---|---|
| 参照対象範囲 | 電車経路テーブル（出発駅・到着駅・運賃） |
| 参照方針 | 交通費計算ツール（TOOL-001）が出発地・目的地をキーに検索する |
| メタデータ | 改定日 |
| 根拠提示 | 計算根拠（使用した経路・運賃）を提示 |
| 更新検知 | 要件上未定義 |

### DATA-004（固定運賃データ）

| 項目 | 内容 |
|---|---|
| 参照対象範囲 | バス・タクシー・飛行機の交通手段ごとの固定金額 |
| 参照方針 | 交通費計算ツール（TOOL-001）が交通手段をキーに固定運賃を取得する |
| メタデータ | 改定日 |
| 根拠提示 | 計算根拠（適用した固定運賃・交通手段）を提示 |
| 更新検知 | 要件上未定義 |
